## 开发者最好的伙伴：产品说明书

### 1. 产品定位
“开发者最好的伙伴”是一个面向开发者的资源发现与效率工具推荐网站，专注推荐好用的开发工具、效率插件、优质网站和开发资源。

网站的核心目标不是做一个普通导航页，而是帮助开发者更快找到“真正值得用”的工具，并理解这些工具适合什么场景、解决什么问题、如何快速上手。

### 2. 产品目标
1. 推荐好用的开发工具，例如编辑器、终端、接口调试、数据库管理、部署、监控、AI 编程工具等。
2. 推荐提升效率的工具，例如浏览器插件、VS Code 插件、命令行工具、自动化脚本、截图录屏、笔记协作工具等。
3. 推荐好用的网站，例如文档站、设计素材站、接口测试站、代码片段站、学习平台、开源项目导航等。
4. 推荐优质开发资源，例如学习路线、技术文档、模板项目、开源库、工程实践文章、面试资料等。
5. 通过分类、标签、搜索、榜单和专题，让开发者能按场景快速找到合适资源。

### 3. 目标用户
| 用户类型 | 典型需求 |
|----------|----------|
| 初级开发者 | 不知道哪些工具值得用，希望获得可靠推荐和上手说明 |
| 有经验开发者 | 想发现更高效的插件、工具链和工程实践资源 |
| 独立开发者 | 需要快速找到建站、部署、设计、支付、数据分析等工具 |
| 技术团队成员 | 希望沉淀团队常用工具、规范资源和协作工具清单 |
| 技术内容读者 | 想浏览高质量网站、开源项目、教程和精选专题 |

### 4. 核心价值
- **省时间**：减少开发者在搜索、试错、比较工具上的时间成本。
- **更可信**：每个推荐资源都说明适用场景、核心优点、使用门槛和替代方案。
- **按场景找资源**：围绕真实开发任务组织内容，例如“接口调试”“前端动效”“AI 辅助编码”“部署上线”。
- **持续更新**：通过榜单、专题和新资源收录，保持内容新鲜度。
- **可收藏可沉淀**：让用户建立自己的开发资源库。

### 5. 核心功能

#### 5.1 首页
- 展示网站定位和主入口。
- 提供搜索框，支持搜索工具、插件、网站和资源。
- 展示热门分类、最新推荐、编辑精选和本周榜单。
- 通过专题入口引导用户按场景浏览。

#### 5.2 资源分类
资源按类型和使用场景双重分类。

| 一级分类 | 示例内容 |
|----------|----------|
| 开发工具 | IDE、终端、Git 工具、数据库工具、接口调试、部署工具 |
| 效率插件 | VS Code 插件、Chrome 插件、Raycast 扩展、Obsidian 插件 |
| 实用网站 | 在线工具、文档站、素材站、代码生成、格式转换、图标字体 |
| 学习资源 | 技术文档、学习路线、课程、博客、开源项目、面试资料 |
| AI 工具 | AI 编程助手、Prompt 工具、代码解释、测试生成、文档生成 |
| 设计与产品 | 原型工具、配色工具、图标资源、UI 模板、灵感网站 |

#### 5.3 资源详情页
每个资源需要包含以下信息：
- 名称、Logo、官网链接。
- 一句话介绍。
- 适用人群和适用场景。
- 核心亮点。
- 使用成本，例如免费、开源、订阅、买断。
- 支持平台，例如 Web、macOS、Windows、Linux、VS Code、Chrome。
- 上手建议。
- 同类替代工具。
- 用户评分和收藏入口。

#### 5.4 搜索与筛选
- 支持关键词搜索。
- 支持按分类、标签、平台、价格、推荐指数筛选。
- 支持按最新、最热、评分最高、编辑推荐排序。
- 支持常用标签，例如 `前端`、`后端`、`AI`、`效率`、`开源`、`免费`、`设计`、`部署`。

#### 5.5 榜单与专题
榜单用于提高浏览效率，专题用于解决具体场景问题。

推荐专题示例：
- 前端开发必备 VS Code 插件
- 独立开发者上线产品工具箱
- AI 编程效率工具清单
- 后端接口调试与文档工具推荐
- Mac 开发者效率插件合集
- 程序员常用在线工具导航

#### 5.6 收藏与个人资源库
- 用户可以收藏资源。
- 用户可以按自定义分组整理资源。
- 用户可以查看最近浏览和收藏历史。
- 后续可支持用户提交资源、写简短评价。

### 6. 页面规划
| 页面 | 主要内容 |
|------|----------|
| 首页 | 品牌介绍、全站搜索、热门分类、编辑精选、最新资源、专题榜单 |
| 分类页 | 分类说明、筛选器、资源列表、排序 |
| 资源详情页 | 资源介绍、亮点、适用场景、使用成本、替代方案、收藏 |
| 专题页 | 围绕具体开发场景组织资源合集 |
| 榜单页 | 按热度、评分、更新时间展示资源排行 |
| 提交资源页 | 用户提交工具、插件、网站或开发资源 |
| 个人收藏页 | 用户收藏、分组、最近浏览 |

### 7. 内容收录标准
资源需要满足至少一项标准：
1. 能明显提升开发效率。
2. 在某个开发场景中足够实用。
3. 有较好的口碑、活跃度或维护状态。
4. 上手成本合理，说明清晰。
5. 相比同类工具有明确优势。

不建议收录：
- 长期不可访问的网站。
- 明显过时且无维护的工具。
- 广告性质强但缺少实际价值的资源。
- 无法说明适用场景的泛泛链接。

### 8. 首期内容建议
首期建议先收录 80-120 个资源，保证网站上线时有足够内容密度。

| 分类 | 建议数量 |
|------|----------|
| 开发工具 | 20 个 |
| 效率插件 | 25 个 |
| 实用网站 | 25 个 |
| 学习资源 | 20 个 |
| AI 工具 | 15 个 |
| 设计与产品 | 15 个 |

### 9. 视觉风格
- 整体风格：专业、清爽、有技术感，但不要做成普通网址大全。
- 主色建议：深蓝黑、代码绿、电光蓝，突出开发者气质。
- 卡片设计：资源卡片信息要紧凑清晰，突出名称、标签、推荐理由和收藏按钮。
- 首页氛围：可以加入终端、代码片段、插件面板、工具箱等视觉元素。
- 交互体验：搜索和筛选要足够快，资源卡片 hover 时展示更多推荐理由。

### 10. 第一阶段范围
第一阶段先完成可展示、可浏览、可筛选的前端版本：
- 首页。
- 分类资源列表。
- 资源详情页。
- 专题页。
- 榜单页。
- 本地模拟资源数据。

第一阶段暂不做：
- 真实登录系统。
- 真实后台管理。
- 复杂评论系统。
- 付费功能。

### 11. 成功标准
- 用户进入网站后，10 秒内能理解这是一个开发者工具与资源推荐网站。
- 用户能通过搜索或分类快速找到目标资源。
- 每个资源都能说明“它解决什么问题”和“为什么推荐它”。
- 首页、分类页、详情页形成完整浏览闭环。
- 网站内容结构方便后续持续扩展和维护。
